Python Environment Variables 

H:\Home\Hapslab>echo %PYTHONPATH%
E:\Python36

/home/hapslab>echo %PYTHONPATH%
/usr/bin/sw/python36



PYTHONPATH is same as JVM world's CLASSPATH. Ideal place for looking python pachages/modules. In JVM world CLASSPATH usage can vary based on project needs. But PYTHONPATH stays mostly unchanged for that stand alone installation. PYTHONPATH may not be an ideal way for importing python packages across diferent python versions (installations). You may see unexpected or unexplained exceptions, crashes of bugs. To have smooth python development have complete development stack at same version. IF you have legacy code on say python 2.7 that you need to call from new python 3.x code stack. Using third party tools like Canopy etc can smoothen issues pointed earlier. Else PYTHONPATH should be ignored for use.

PYTHONHOME is used as an alternative module search path. Its purpose is to switch between different python installations. But same objective is achieved easily without causing any issues via PATH hence PYTHONPATH is recommended instead. Also using PYTHONHOME you can run into.

PATH variable keeps binaries located in installation directory of Python or JDK (Java Development Kit) or any JVM language like Scala , Groovy , Clojure etc.


>>> import os
>>> print ( os.environ.get('PYTHONPATH') )

Python Packages, Interfaces, Modules, Classes, & Functions
#WIP 


Packages in python are namespaces which can contain multiple packages, classes and modules themselves. This collection of package can also be called an API. A package in Python is a directory which always contains a special file called __init__.py. File can be empty indicating that the directory it contains is a Python package. Packages can be imported the same way a module can be imported. We can have different directories dedicated to specific packages for better organization of code stack across different application areas.



Modules in python are simply files with the .py extension. Each file implement a or set of functions. Modules can be imported from other modules using the import command. We can import any of the standard list of built-in python modules in the standard library. You can also glossary over Module and 
Package


Classes just like in any OOP languages like C++, Modula-3 etc help to define associated methods & variables primarily. Class get usually mapped to the Entity in the data world. Entity forms the building block of E-R Diagram/ Data Modelling for data design/modelling.
Python classes provide all standard features of Object Oriented Programming: inheritance, methods overriding, etc. As is true for modules, python classes from the dynamic nature of Python: they are created at runtime, and can be modified further after creation.



Methods are part of classes just like in any OOP languages like C++, Modula-3 etc. Some difference do apply to variables, objects, interfaces (none in python) etc. Methods & Functions has some interesting differences. Also Methods can be functions but functions cannot be methods. 



Functions in python are usually contained in Modules. Python has Built-in Functions too like many other languages. Python also has extensive Functional Programming Modules that can help in better exploration and interactive analysis of data.
Scala Packages, Interfaces, Modules, Classes, & Functions 
Packages in Scala is a special object which defines a set of member classes, objects and packages. Packages can be imported same as in python or java, using import package_name or import package_name.member.



There are no Modules as such in Scala. But scala objects and path dependent types work like Modules



Classes in Scala can contain methods, values, variables, types, objects, traits, and classes which are collectively called members.



Everything is an object in Scala including numbers, functions. Object in Scala is an instance of the class. Methods can work on objects.



Java 8 Interfaces and Scala Traits have lot of similarities. Traits are used to share interfaces and fields between classes.



Methods in Scala (as in Java) are part of a class. It has a name, signature, optionally some annotations. In Scala it is possible to nest function definations called Nested Methods



Functions in Scala is a complete object. Function object has methods as an instance of a class that implements one of the traits of the class. Functions in Scala that can take another function as an argument are called Higher-order-functions.
